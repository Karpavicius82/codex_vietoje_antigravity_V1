# API Change Skill

Use this skill when changing HTTP APIs, service contracts, schemas, or integrations.

## Steps

1. Locate route/controller, service layer, schema/types, and tests.
2. Check backward compatibility and validation behavior.
3. Update implementation and contract docs if present.
4. Add or update tests for request, response, errors, and auth behavior.
5. Run targeted tests.

## Output

- Contract changes.
- Compatibility notes.
- Tests run.
